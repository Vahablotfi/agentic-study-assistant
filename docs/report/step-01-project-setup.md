# Step 01 — Project Setup

The project repository was created using Git and GitHub to support version control, documentation, and reproducibility.

The initial structure separates source code, course materials, prompts, workflows, graph scripts, MCP server files, screenshots, and report notes.

This structure supports the exam requirement of documenting the architecture, implementation, and use of the AI agent system.

## Repository Structure

- `lecture-materials/` stores course documents used as the RAG corpus.
- `rag/` stores code related to chunking, embeddings, and retrieval.
- `graph/` stores Neo4j and Cypher-related files.
- `workflows/n8n/` stores exported n8n workflows.
- `prompts/` stores custom prompts and skills.
- `mcp/` stores MCP server code.
- `telegram/` stores interface-related code.
- `docs/report/` stores report notes written during development.

## Reflection

Using GitHub from the beginning makes the project easier to reproduce and provides a clear development history for the final report and oral exam.
